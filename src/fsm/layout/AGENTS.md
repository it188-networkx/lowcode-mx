## 目录职责

本目录存放 fsm 命名空间的 18 个布局 JSON 文件。
Layout 文件只存储 blockID 引用和 xywh 坐标，Block 完整定义在 Page JSON 中。

---

## 文件命名格式

```
{LayoutTitle}_{layoutID}.json
```

---

## 三种 Layout 类型

| 类型 | handle | visibility.expression | 按钮特征 |
|------|--------|----------------------|---------|
| 列表页 | primary | "" | new + back 启用 |
| 查看 | "" 或 isView | isView | back + edit 启用 |
| 新建 | primary | isCreate | back + submit 启用 |
| 编辑 | "" | isEdit | back + submit 启用 |

---

## 操作规则

- **改 Block 位置**：修改 Layout JSON 中对应 blockID 的 `xywh` 数组
- **添加/删除 Block**：必须同步修改对应的 Page JSON
- **修改按钮**：修改 Layout JSON 的 `config.buttons` 或 `config.actions`
- **新建 Layout**：必须通过 Sonyflake 生成 layoutID；创建 Page 时必须同时创建 Layout

操作前必读：`../../../../../lowcode-base/process/layout/`
